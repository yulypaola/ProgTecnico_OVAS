Estructuras de control secuenciales y condicionales

En programación, las estructuras de control son como el mapa que guía a un programa. Sin estas estructuras, el programa solo podría ejecutar instrucciones una tras otra, sin tomar decisiones ni reaccionar a distintas situaciones. Con esta guía aprenderás paso a paso cómo funcionan las secuencias y las estructuras condicionales, para que puedas usarlas sin miedo y con confianza.

Estructuras de control secuenciales ＋

Las estructuras de control secuenciales Representan el flujo natural de la lógica: una acción ocurre después de otra. son la forma más básica de organización en un algoritmo o programa. En ellas, las instrucciones se ejecutan una tras otra, en el mismo orden en que fueron escritas, sin condiciones ni repeticiones.

Estructuras de control. Aunque son limitadas, siempre estarán presentes, ya sea de manera independiente o combinadas con estructuras condicionales y repetitivas para resolver problemas más complejos. — Las estructuras de control secuenciales constituyen el pilar de la programación, pues aseguran que las instrucciones se ejecuten de forma ordenada y lógica.

Las estructuras condicionales permiten controlar el flujo de un programa al tomar decisiones basadas en condiciones lógicas. Estas pueden ser simples Ejecutan una acción si se cumple la condición , dobles Definen un camino cuando la condición es verdadera y otro cuando es falsa o múltiples Permiten elegir entre varios casos posibles .

Condicional Simple (Si) ＋

El condicional simple (si) ejecuta un bloque de instrucciones solo si se cumple una condición. Si no se cumple, continúa con la siguiente instrucción sin ejecutar el bloque.

Condicional compuesto o doble (Si - sino) ＋

Cuando existen dos caminos posibles, usamos si-sino (if–else). Si la condición es verdadera, ejecuta el primer bloque; en caso contrario, ejecuta el bloque del sino.

Condicionales múltiples ＋

Los condicionales múltiples permiten tomar decisiones cuando existen varias condiciones posibles. Se pueden implementar de dos formas:

  • Mediante múltiples condicionales SI anidados Donde cada condición se evalúa de forma secuencial. .
  • Mediante la estructura SEGÚN Facilita la lectura cuando se compara una misma variable contra diferentes valores. , que mejora la claridad del código.
Condicionales múltiples con SI ＋

Los condicionales múltiples con SI permiten evaluar diferentes escenarios de manera secuencial. En esta estructura, si la primera condición no se cumple, el programa pasa a revisar la siguiente y así sucesivamente, hasta encontrar una que sea verdadera o ejecutar el bloque final de lo contrario.

Esta forma de anidar condicionales es útil cuando se requiere comparar varias posibilidades que no pueden resolverse con una sola instrucción condicional.

Condicionales múltiples con Según ＋

El condicional SEGÚN Conocido en otros lenguajes como switch permite evaluar una sola variable y compararla contra distintos valores posibles. Según el valor que tome, se ejecuta un bloque de acciones.

Ejercicios Prácticos con estructuras condicionales ＋

Las estructuras de control condicionales permiten que un programa tome decisiones según se cumpla o no una condición lógica. Estas condiciones se expresan como proposiciones que pueden

Enunciado 1. Un cine necesita calcular el valor a cobrar por tiquete. Si el cliente es menor de 12 años recibe 20% de descuento sobre el precio base.

Enunciado 2. Un restaurante aplica descuentos según el día de la semana: (lunes (1): 10%, martes (2): 15%, miércoles (3): 20%, Otros días: sin descuento). Calcular el total a pagar para una cuenta dada.

Actividades

Situación 1

Escriba el pseudocódigo para un programa que determine si una persona es mayor de edad (18 años o más).

Situación 2

Un supermercado aplica descuentos así: Si el cliente compra más de 10 productos, obtiene 15% de descuento. Si compra entre 5 y 10, obtiene 10%. Si compra menos de 5, no tiene descuento. Diseñe el pseudocódigo con condicionales múltiples para este caso.

Situación 3

Usando la estructura SEGÚN, diseñe un pseudocódigo que muestre el día de la semana según un número ingresado (1 = lunes, 2 = martes, …, 7 = domingo).
