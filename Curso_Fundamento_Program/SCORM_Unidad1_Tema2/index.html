Identificar antes de programar 👀

Antes de escribir código, un buen programador define con precisión el problema. En este tema aprenderás a delimitar qué se debe resolver, analizar datos de entrada/salida y detectar restricciones para proponer soluciones lógicas.

Unidad 1 · Tema 2 — Identificación y análisis

Ruta de análisis 🧭

  1. Comprender el contexto: quién lo necesita y para qué.
  2. Delimitar alcance: qué incluye y qué no incluye.
  3. Definir entradas/salidas: datos que recibe y resultados esperados.
  4. Listar restricciones: reglas, validaciones, tiempos, formatos.
  5. Criterios de éxito: cómo sabrás que funciona.
Consejo: anota los supuestos y valida con tu docente o cliente.

Voces del equipo 💡

Analista de Requerimientos

"Mi tarea es entender el problema real. Un buen análisis evita retrabajo y soluciones equivocadas."

QA / Tester

"Yo traduzco el análisis en casos de prueba. Si las salidas no están claras, no puedo validar el éxito."

Marco del problema 🧩

Entradas

  • Datos proporcionados por el usuario o sistema.
  • Formato y rango válidos.

Proceso

  • Transformaciones y reglas de negocio.
  • Algoritmos aplicados.

Salidas

  • Resultados visibles o acciones.
  • Métricas para verificar éxito.

Ejemplos prácticos 📄

Ejemplo 1: Salario Neto

  • Entradas: horas, tarifa, % deducciones.
  • Proceso: (horas×tarifa)−deducciones.
  • Salida: salario neto.
  • Restricciones: horas ≥ 0; tarifa ≥ SMMLV/h; % 0–100.

Ejemplo 2: Validador de Correo

  • Entradas: email.
  • Proceso: comprobar formato y existencia.
  • Salida: “válido” / “inválido”.
  • Restricciones: dominio permitido, sin espacios.
