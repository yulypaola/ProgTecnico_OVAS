🚀 La Programación. Tu Herramienta Principal

Programar es, en esencia, darle instrucciones claras a un computador para que realice una tarea específica. Es el lenguaje que usamos para comunicarnos con la tecnología que nos rodea.

¿Por qué es tan importante para un Programador Web?

  • Todo es Programación: Desde el botón que un usuario presiona para iniciar sesión hasta el sistema que procesa una compra en línea, todo funciona con código. Sin programación, una web sería solo un documento de texto estático y aburrido.
  • Resolución de Problemas: Ser programador es ser un "solucionador" de problemas profesional. Aprenderás a tomar un gran desafío (como "crear una red social") y dividirlo en partes pequeñas y manejables que puedes resolver con código.
  • Desarrolla tu Lógica: La programación te obliga a pensar de forma estructurada y lógica. Esta habilidad no solo te servirá para programar, sino en todos los aspectos de tu vida.
  • Innovación y Creatividad: Con la programación, el único límite es tu imaginación. Puedes crear desde un simple blog personal hasta una compleja aplicación web interactiva que ayude a miles de personas.

🧠 ¿Qué es un Algoritmo?

Antes de escribir una sola línea de código, necesitas un algoritmo.

Un algoritmo es simplemente una secuencia de pasos ordenados y finitos para resolver un problema. ¡Usas algoritmos todos los días sin darte cuenta!

Ejemplo cotidiano: Hacer Arroz con Coco

  1. INICIO
  2. Rallar el coco.
  3. Exprimir el coco para sacar la primera leche (el titoté).
  4. Poner la primera leche en el caldero a fuego alto hasta que salga el aceite y se dore.
  5. Agregar el arroz y sofreír.
  6. Añadir el resto de la leche de coco y el agua necesaria.
  7. Agregar sal y azúcar al gusto.
  8. Esperar a que se seque y tapar a fuego bajo.
  9. Servir cuando esté listo.
  10. FIN

Si te saltas un paso o cambias el orden, el resultado no será el mismo. En programación, ocurre exactamente lo mismo. El algoritmo es tu plan de acción antes de programar.

📋 Ejemplos de Algoritmos en la Web

Ejemplo 1: Inicio de Sesión (Login)

Problema: Verificar si un usuario puede acceder a su cuenta.

  1. INICIO
  2. Pedir al usuario su correo y contraseña.
  3. Buscar en la base de datos si el correo existe.
  4. SI el correo existe, comparar la contraseña ingresada con la guardada.
  5.     SI las contraseñas coinciden, ENTONCES mostrarle su página de perfil.
  6.     SI NO coinciden, ENTONCES mostrar un mensaje de "Contraseña incorrecta".
  7. SI NO el correo no existe, ENTONCES mostrar un mensaje de "Usuario no encontrado".
  8. FIN

Ejemplo 2: Agregar producto al carrito

Problema: Añadir un artículo que el usuario quiere comprar.

  1. INICIO
  2. El usuario hace clic en el botón "Añadir al carrito".
  3. Identificar el producto seleccionado (nombre, precio, ID).
  4. Verificar si ya existe un carrito para ese usuario.
  5. SI existe, agregar el producto al carrito existente.
  6. SI NO existe, crear un nuevo carrito y agregar el producto.
  7. Actualizar el ícono del carrito con la nueva cantidad de productos.
  8. Mostrar un mensaje de confirmación: "¡Producto añadido!".
  9. FIN

✍️ ¡Ahora a Practicar!

¡Es tu turno de pensar como programador! No necesitas escribir código, solo describe el algoritmo (los pasos) para resolver estos problemas.

1. Formulario de Contacto

Un usuario llena un formulario en una página web con su nombre, email y un mensaje, y luego presiona "Enviar". Describe el algoritmo de lo que debe pasar después. ¿Qué verificaciones harías? ¿Qué mensaje le mostrarías al usuario al final?

2. Buscador de Canciones

Piensa en Spotify. Un usuario escribe "Joe Arroyo" en la barra de búsqueda y presiona Enter. Describe el algoritmo que debería seguir la plataforma para mostrarle los resultados.

3. Botón de "Seguir"

Describe el algoritmo para un botón de "Seguir" en un perfil de Instagram. ¿Qué pasa la primera vez que un usuario le da clic? ¿Y qué pasa si le vuelve a dar clic (para dejar de seguir)?
